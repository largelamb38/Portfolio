Responsive Redesign

Redesigning an online tennis shop based in Hong Kong

Content/Problem

As a varsity tennis player from Hong Kong, I’ve frequently purchased tennis apparel/shoes from Tennis Star HK and since “Jeff Tennis” (the owner of the shop who is not a CS major) developed his own website, I’ve noticed that there were many issues with the website that could be fixed to improve user experience.

Link to original website: Tennis Star HK

URL: https://www.tennisstarhk.com/adidas-men-s-shoes

Research/Feedback


Problems I've Observed


  1. Branding and memorability issue as shop name and location are not on front header
  2. When users click on a product to try and purchase, they are only shown an enlarged photo of the product (nothing to add to cart or steps on how to purchase)
  3. There is no shopping cart to add items users want to purchase
  4. The purchasing process is not clearly labelled
  5. There is no sort function for products (sorting based on price or popularity)
  6. Shoes of all brands should be displayed as it is inconvenient for users to have to go through different brand pages just to find the shoes they want to buy
  7. There is no rating system for products, so users are not able to gauge product quality
  8. The only language available is English which makes the website hard to use, especially considering that most Hong Kong people can only read simplified/traditional Chinese

Accessibility issues:



  1. There were 22 contrast errors all pointing to the text and its background which make it difficult for users (especially color blind users) to read. The color themes and schemes of this website need to be changed.
  2. The website is only available in English. Considering that a large percentage of the Hong Kong population have Cantonese as their first language, it would be wise to add an optionality for simplified/traditional Chinese to increase accessibility
  3. There are a large number of headers of equal size which make elements of the website and their importance hard to comprehend, especially for blind users who rely on hearing software

Conclusion



This project was a great opportunity for me to put my CSS and HTML skills to the test. Something surprising to me was the importance of a color theme. I didn't realize that the colors selected on my website needed to have a specific amount of contrast for colorblind readers. Also, the website needed to be formatted very precisely so that it is easy for users to understand how to use it.
